## Summary

The AML parser accepts inheritance from anonymous objects (e.g., `Derived:Base` where `Base { x := 10 }` was declared without `:=`), violating the semantic contract that anonymous objects are immutable/read-only. This creates a logical inconsistency: if anonymous objects are truly immutable, they should not be inheritable.

## Environment

- **anvil version:** v0.1.0-alpha (post FR-2604-anvl-002 merge)
- **Affected components:** parser.c, resolver.c
- **Test suite:** test/unit/test_resolver.c (CM01-CM10)

## Reproduction

**Current behavior (INCORRECT):**

```aml
#!aml

Base { x := 10 }                # Anonymous object (supposedly immutable)
Derived:Base := { y := 20 }     # Parser accepts this, resolver processes it
```

**Result:** No error, inheritance succeeds, `Derived` gets merged fields from `Base`.

**Test code (from test_resolver.c):**

```c
#define CM_BASE_DERIVED \
   "#!aml\n"            \
   "Base { x := 10 }\n" \
   "Derived:Base := { y := 20 }\n"

// Test passes - anonymous Base is successfully inherited
static void test_get_base_index_with_inheritance(void) {
   context ctx = build_context(CM_BASE_DERIVED);
   Assert.isNotNull(ctx, "Context should build");  // ✅ Passes
   
   anvl_node_state_t *state = anvl_resolver_build_state(ctx, ctx_src(ctx));
   Assert.isNotNull(state, "build_state should succeed");  // ✅ Passes
   
   usize base_idx = anvl_node_state_get_base_index(state, 1);
   Assert.isTrue(base_idx == 0, "Derived should have base index 0");  // ✅ Passes
   // ...
}
```

All CM01-CM10 tests use anonymous base objects and currently pass.

## Expected Behavior

**Option 1 (RECOMMENDED): Parser should reject inheritance from anonymous objects**

```aml
#!aml

Base { x := 10 }                # Anonymous (immutable)
Derived:Base := { y := 20 }     # ❌ PARSER ERROR: Cannot inherit from anonymous object
```

**Error message:** `ANVL_ERR_CANNOT_INHERIT_FROM_ANONYMOUS` at resolver time when base lookup finds an `ANVL_ANON_OBJECT` statement.

**Correct usage:**
```aml
#!aml

# For inheritance, use explicit assignment
Base := { x := 10 }             # Regular assignment (inheritable)
Derived:Base := { y := 20 }     # ✅ Valid

# OR: Use anonymous for standalone configs
Config { timeout := 30 }        # Anonymous (not inherited from)
```

## Root Cause

### Parser Implementation (src/core/parser.c)

The `parse_anon_object()` function marks statements as `ANVL_ANON_OBJECT`:

```c
stmt->meta[STMT_META_TYPE] = (usize)ANVL_ANON_OBJECT;
```

However, when `parse_statement()` processes inheritance (`:Base` syntax), it performs an identifier lookup without checking if the base statement is anonymous:

```c
// parse_statement() - lines ~220-250
if (si_match_length(s, ":", 1) == 1 && si_peek_offset(s, 1) != '=') {
    si_consume(s, 1);
    si_skip_whitespace_and_comments(s);
    if (!read_identifier(p, &base_pos, &base_len))
        return false;
    
    // Allocate base_meta (SHOULD CHECK if base is anonymous here!)
    base_meta = ci_new_base_meta(p->ctx);
    // ...
}
```

The base identifier is stored in `stmt->base_meta` without validation.

### Resolver Implementation (src/resolver/resolver.c)

When building resolver state, the code looks up base statements by name without checking type:

```c
// anvl_node_state_get_base_index() - line ~595
usize base_stmt_idx = id_map_lookup(&state->id_to_idx, raw,
                                    stmt->base_meta->pos,
                                    stmt->base_meta->len);

if (base_stmt_idx == (usize)-1) {
    anvl_error_set(ANVL_ERR_RESOLVER_MISSING_BASE, ...);
    return (usize)-1;
}

return base_stmt_idx;  // ← Returns index without type check
```

**No validation** that `Context.get_statement(ctx, base_stmt_idx)->meta[STMT_META_TYPE]` is **not** `ANVL_ANON_OBJECT`.

## Impact

### Semantic Violation

- **Design inconsistency:** Anonymous objects claim to be "read-only" but are fully mutable via inheritance
- **User confusion:** `Base { }` syntax implies immutability, but `Derived:Base` works
- **Documentation conflict:** If anonymous means immutable, inheritance contradicts this

### Affected Components

- **Parser:** Needs validation to reject `Derived:AnonBase` syntax at parse time
- **Resolver:** Should error during `anvl_resolver_build_state()` if base is anonymous
- **Tests:** CM01-CM10 all use anonymous base objects (need rewrite)
- **Documentation:** EX-2604-sigma-cli-007 documents this incorrect usage

### Phase 1: Add Error Code

### Phase 2: Resolver Validation

**`src/resolver/resolver.c` in `anvl_node_state_get_base_index()`:**

```c
usize anvl_node_state_get_base_index(anvl_node_state_t *state, usize stmt_idx) {
    // ... existing code ...
    
    usize base_stmt_idx = id_map_lookup(&state->id_to_idx, raw,
                                        stmt->base_meta->pos,
                                        stmt->base_meta->len);
    
    if (base_stmt_idx == (usize)-1) {
        anvl_error_set(ANVL_ERR_RESOLVER_MISSING_BASE, ...);
        return (usize)-1;
    }
    
    // NEW: Check if base is anonymous
    statement base_stmt = Context.get_statement(state->ctx, base_stmt_idx);
    if (base_stmt && 
        (anvl_stmt_type)base_stmt->meta[STMT_META_TYPE] == ANVL_ANON_OBJECT) {
        anvl_error_set(
            ANVL_ERR_CANNOT_INHERIT_FROM_ANONYMOUS,
            "cannot inherit from anonymous object",
            stmt->base_meta->pos,
            stmt->base_meta->len,
            __FILE__
        );
        return (usize)-1;
    }
    
    return base_stmt_idx;
}
```

**Breaking semantic change:** Code that previously inherited from anonymous objects will now fail with error 4052. This is intentional and enforces the immutability contract.

Base := { x := 10 }          # Explicit assignment (inheritable)
Derived:Base := { y := 20 }  # ✅ Valid
