## Summary

anvil.o (built March 23) causes **segmentation fault** in `_anvil_init()` when used with sigma.core.o rebuilt for FR-005 (ctrl field added to sc_alloc_use_t at offset 0). anvil needs a clean rebuild with the updated sigma.core headers.

## Environment

- **anvil.o**: Mar 23 17:47 (OLD - built before FR-005)
- **sigma.core.o**: Mar 26 21:31 (NEW - FR-005 + FR-006)
- **sigma.memory.o**: Mar 26 21:33 (NEW - rebuilt)
- **sigma.collections.o**: Mar 26 21:02 (NEW - rebuilt)

## Reproduction

With current library versions:
```bash
cd /home/badkraft/repos/sigma.test
cbuild clean && cbuild
./rtest asserts
```

**Expected:** Test runs successfully  
**Actual:** Segmentation fault during module initialization

Crash occurs at NULL pointer (0x0) during `_anvil_init()` module initialization, before any test code executes.

## Root Cause

**ABI incompatibility:** anvil.o was compiled with old sigma.core headers (before FR-005). If anvil uses or interacts with `sc_alloc_use_t` structures, function pointers, or other sigma.core types, the structure layout mismatch causes undefined behavior.

**FR-005 changes:**
- Added `ctrl` field as **first member** of `sc_alloc_use_t` (at offset 0)
- Shifted all subsequent members by 8 bytes
- Any code compiled with old headers interprets structure offsets incorrectly

## Resolution Required

**Simple rebuild:**
```bash
cd /home/badkraft/repos/anvil
cbuild clean
cbuild
cpkg anvil
sudo cp package/anvil.o /usr/local/packages/
```

No source changes needed - just recompile with current sigma.core headers from `/usr/local/include/sigma.core/`.
