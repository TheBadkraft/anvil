# Document Standards — anvil
- Standards: [q-or/DOC_STANDARDS.md](../q-or/DOC_STANDARDS.md)
- Registry:  [q-or/registry.anvl](../q-or/registry.anvl)
- This project's doc root: `anvil/bug-reports/`, `anvil/feature-reqs/`, `anvil/tickets/`
- Cross-project link syntax: `../other-project/subdir/filename.md`
- To publish a doc for cross-project use: add entry to registry.anvl first
- ID prefix for new bugs: `BR-{YYMM}-anvil-{seq:03d}`
- ID prefix for new features: `FR-{YYMM}-anvil-{seq:03d}`
## Docs
- Register a new document: `python3 ../tools/doc-register.py --id <ID> --type <type> --owner anvil --path "anvil/subdir/file.md" --title "..." --status open --tags "tag1,tag2"`
- `--type` values: `bug-report`, `feature-req`, `ticket`, `use-case`, `backlog`
- Or edit [q-or/registry.anvl](../q-or/registry.anvl) directly; full conventions: [q-or/DOC_STANDARDS.md](../q-or/DOC_STANDARDS.md)
- Dependency state: check [q-or/repo.anvl](../q-or/repo.anvl) before starting feature work
## Ops
### Build (standalone — no external dependencies)
```bash
# Core parser only
gcc -std=c2x -I./include $(grep -v 'operators\|symbols\|types' <(ls src/core/*.c) src/utilities/*.c) -c

# Using config.sh source arrays
source config.sh
gcc -std=c2x -I./include "${ANVIL_SOURCES[@]}" -c

# Default lib (parser + resolver + vars)
gcc -std=c2x -I./include src/core/anvil_impl.c src/core/context.c \
    src/core/errors.c src/core/memory.c src/core/parser.c \
    src/core/strings.c src/utilities/utils.c \
    src/resolver/resolver.c src/vars/vars.c -c
```

### Test (TestBit — zero external dependencies)
```bash
cd test/unit
make all          # build all test binaries
make valgrind     # build + run all under valgrind
make clean        # remove bin/

# Individual test binaries in test/unit/bin/
./bin/test_scalar_quotes      # QS01-QS06: quote stripping
./bin/test_amp_collections    # AM01-AM07: AMP arrays/tuples
./bin/test_interp_string      # IS01-IS08: interpolated strings
./bin/test_using              # US01-US07: using declarations
./bin/test_anon_block_attrs   # AB01-AB07: anonymous block @[attrs]
./bin/test_resolver           # RS01-RS12: inheritance resolver
./bin/test_schema             # SC01-SC10: schema validation
```
