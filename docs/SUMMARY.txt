MEMORY LEAK FIXES (3 CRITICAL)
================================================================================

LEAK #1: Line 178-180 (Missing := operator)
  • Issue: base_meta and attr_meta not disposed on missing :=
  • Fix: Added conditional disposal before error return
  • Impact: Prevents leak in malformed statements

LEAK #2: Line 196 (Attributes on scalar)
  • Issue: val not disposed when attributes applied to scalar
  • Fix: Added val disposal in error path
  • Impact: Prevents leak in invalid attribute cases

LEAK #3: Line 211 (value_meta allocation)
  • Issue: val not disposed when value_meta allocation fails (OOM)
  • Fix: Added val disposal in memory failure path
  • Impact: Prevents leak during resource exhaustion

All fixes verified with test cases. Pattern documented for future use.

DESIGN PATTERNS DOCUMENTED
================================================================================

1. Parser Context Pattern
   └─ Single parameter carries state + context

2. Temporary vs. Owned Pattern
   └─ Explicit lifecycle distinction for allocations

3. Cleanup-on-Error Pattern
   └─ Every error path disposes allocations

4. Conditional Disposal Pattern
   └─ Safe NULL checks before Memory.dispose()

5. Zero-Copy Spans Pattern
   └─ Position + length instead of string copies

6. Metadata Extraction Pattern
   └─ Separate parsing from metadata organization

7. Naming Conventions
   └─ Consistent prefixes (parse_, si_, ci_, etc.)

8. Forward Declaration Pattern
   └─ Enable mutual recursion at file top

9. Error State Globalization
   └─ Single global error state (anvil.c)

10. Metadata Buffer Pattern
    └─ Self-contained per-statement metadata
